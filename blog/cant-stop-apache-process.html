Can't Stop Apache Process.

Restarting Web Server Apache2 [fail]. Please Review the Situation By Hand.

16 Apr 2015 in Ubuntu | by Kalob Taulien

If your website is down there are a number of different possible causes. Here are just a couple of the more common problems that can cause a website to not load.

  1. Your domain name provider might be down or experiencing a problem.
  2. A DNS problem (if you rebooted an Amazon EC2 instance, remember to re-associate your Elastic IP)
  3. A network problem
  4. A service your running has crashed (MySQL, Apache, etc)
  5. The entire server is down.

In this scenario we're facing the #4 problem, A server your running has crashed. Although I say "crashed" it could just be hung up on something. This time, we're talking about the dreaded apache2 process hangup. When you try to restart Apache using sudo service apache2 restart you might see an error like the one below:

Restarting web server apache2 [fail]
There are processes named 'apache2' running which do not match your pid file which are left untouched in the name of safety, Please review the situation by hand.

When this happens, you can run a CLI command in your Ubuntu terminal to get the process ID's that Apache is running. I;ve only ever noticed one process ID when getting this error but I'm sure you can get multiples.

To fix this apache problem, we need to kill the process ID and restart Apache.

  1. In your terminal type pidof apache2 and you will get a list of process ID's (or just one). Something like this:
    ubuntu: pidof apache2
    1118
  2. Now kill this process by typing sudo kill -9 1118
  3. Restart Apache with sudo service apache2 restart
Your server should be up and runinng now. If it isn't, then this was not your problem.

Additionally, you can kill every Apache2 process at the same time by typing sudo killall -9 apache2
