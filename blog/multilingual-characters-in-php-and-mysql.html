Multilingual Characters In PHP And MySQL

How to Store And Retrieve Foreign Letters.

17 Apr 2015 in PHP | by Kalob Taulien

Here's a quick guide on how to use multilingual and foreign characters in MySQL and access them in PHP. 

First and foremost, if you're using the MySQL() PHP function - start using MySQLi() or PDO() (I prefer PDO so this article is going tackle this problem with a bit of PDO)

First, you'll want to set a few settings in your PHP, preferably before everything else you're about to write. Possibly at the top of your config file. 

Inside your PHP file add: 

<?php          
    mb_internal_encoding('UTF-8');  // Tell PHP to use UTF-8 inside PHP
    mb_http_output('UTF-8');    // Tell PHP to push UTF-8 through the buffer to the browsers
    header('Content-Type: text/html; charset=UTF-8'); // Tell the browser we're using the UTF-8 character set
?>
Inside you're HTML <head> add: 

<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />

To connect to your MySQL database use one of the following:
A) PDO: 

<?php
    $db = new \PDO( 'mysql:charset=utf8mb4;mysql:host=<localhost>;dbname=<db_name>', '<user_name>', '<pass_phrase>' );
?>

B) MYSQLI 

<?php
    $mysqli->set_charset('utf8mb4'); // OOP Method
    mysqli_set_charset($link, 'utf8mb4');  // Procedural Method   
?>

One last thing. If you run all of this and find that your MySQL database is not storing the correct characters, double check that your tables' collation is set to utf8mb4_unicode_ci (that means UTF-8 and is case insensitive).

Now when you INSERT or SELECT from your MySQL Database, you'll see all the proper characters. 

Essentially what you've told your application is:
  1. Use UTF-8 Encoding inside the server-side scripting
  2. Use UTF-8 Encoding inside your MySQL application. 
  3. Tell the browser to force UTF-8 encoding via HTML and PHP
